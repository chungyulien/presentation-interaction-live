# 2026-05-28 工作流程紀錄

專案：互動堂 Live  
目的：建立可實際連網操作的簡報即時互動網站，支援講者端、觀眾端、投影模式、選擇題與文字雲。

## 1. 需求整理

今天先依據系統規格書確認核心需求：

- 講者可建立互動房間，取得 6 碼 PIN 與 QR Code。
- 觀眾免登入，用 PIN 或 QR Code 加入。
- 支援選擇題即時投票。
- 支援文字雲即時送出、統計與顯示。
- 提供乾淨的投影模式，方便搭配簡報投影。
- 需要能上架到網路，讓不同裝置實際連線操作。

## 2. 技術決策

原本規格建議 React/Vue、Socket.io、Redis 或 MongoDB。實作時因本機沒有可用 npm 環境，改採零安裝版本：

- 前端：原生 HTML、CSS、JavaScript SPA。
- 後端：Node.js 內建 HTTP Server。
- 即時同步：原生 WebSocket。
- 狀態儲存：Node.js 記憶體，適合課堂現場原型。
- QR Code：伺服器端產生 SVG，不依賴外部 CDN。

此做法保留即時互動能力，也讓專案能直接用 `node server/index.js` 啟動。

## 4. 上架準備

因 GitHub Pages 只能放靜態網站，不能跑 WebSocket 後端，因此選擇 Render 作為正式免費上架方案，Cloudflare Quick Tunnel 作為臨時公開測試方案。

新增上架相關檔案：

- `render.yaml`：Render Blueprint 設定。
- `Procfile`：支援部分 Node.js 主機。
- `DEPLOYMENT.md`：上架說明。
- `start-public-preview.ps1`：建立 Cloudflare Quick Tunnel 臨時公開網址。

也調整伺服器：

- 支援雲端平台提供的 `PORT`。
- 監聽 `0.0.0.0`，讓雲端容器可連入。
- 新增 `/api/health` 健康檢查。
- 新增 WebSocket heartbeat，降低閒置斷線機率。
- 對 HTML、JS、CSS 加上 `no-cache`，減少更新後吃到舊版的機會。

## 9. 音效調整

完成文字雲音效：

- 觀眾端送出文字成功時播放短提示音。
- 講者操作端收到觀眾文字時播放較明顯的三音提示。
- 投影端收到文字時也沿用操作端提示邏輯。
- 第一次點擊建立房間、發布題目或其他操作時會解鎖瀏覽器音訊。
- 若音訊尚未解鎖，操作端收到的提示會先排隊，待下一次點擊後補播。

- 可用臨時公開網址測試。

仍需注意：

- 目前房間資料存在伺服器記憶體中，服務重啟後房間會消失。

- Cloudflare Quick Tunnel 是臨時網址，不適合作為長期正式網址。
- 若未來要保存課後紀錄，可接 MongoDB Atlas、Render Postgres 或其他資料庫。
